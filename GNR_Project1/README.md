## Pipeline Overview

This project solves the spatial map reconstruction and evaluation task using a 3-phase hybrid architecture:

1. **Computer Vision (Stitching):** Utilizes Perceptual Hashing and a Directed Acyclic Graph (DAG) to dynamically reconstruct the shredded map geometry without relying on arbitrary resizing.
2. **Spatial Indexing (OCR):** Passes the stitched canvas through PaddleOCR to extract text bounding boxes, creating a mathematical coordinate grid of place names.
3. **Hybrid VQA Engine:** Analyzes questions dynamically. It uses mathematical coordinate calculations for spatial questions (e.g., "closest to", "northernmost") and routes complex visual reasoning tasks to a localized instance of `Qwen2-VL-7B-Instruct`. A strict confidence threshold (0.20) is enforced to minimize the -0.25 incorrect answer penalty.

**Expected Runtime:** Well within the 1-hour competition limit on an NVIDIA L40s GPU.

## Setup Phase (Internet Required)

Before running offline inference, you must initialize the environment and download the required weights. Open your terminal in the root directory of this project and run:

```bash
bash setup.bash
```

*Note: This script handles Conda environment creation, dependency installation, and securely caching the ~15GB Qwen2-VL weights and PaddleOCR models for offline use.*

## Inference Phase (Strict Offline Mode)

### 1. Expected Test Directory Structure

To evaluate this pipeline, the `--test_dir` argument must point to an absolute directory containing the hidden evaluation data.

Our script expects the TA's secret evaluation directory to be structured exactly like this:

```text
<absolute_path_to_test_dir>/
├── patches/                 # Folder containing all shredded map images
│   ├── patch_0.png
│   ├── patch_1.png
│   └── ...
└── test.csv                 # The MCQ question file
```

### 2. Execution Command

Once the test directory is prepared, activate the environment and run the pipeline:

```bash
conda activate gnr_project_env
python inference.py --test_dir <absolute_path_to_test_dir>
```

### 3. Output

The pipeline will safely generate the final answers in the **current working directory** (protecting the integrity of the potentially read-only test directory):

* `submission.csv`: Contains the final parsed answers (Format: `id`, `question_num`, `option`).
* `stitched_map.png`: The visually reconstructed map canvas generated during Phase 1.
